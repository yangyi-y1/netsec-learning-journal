## OSI 七层模型速记

口诀：**"物链网传会表应"**（从下往上 1→7）

| 层 | 名称 | 作用 | 协议/设备 |
|----|------|------|-----------|
| 7 | 应用层 | 用户直接面对 | HTTP/DNS/SSH |
| 6 | 表示层 | 加密、格式转换 | TLS/SSL |
| 5 | 会话层 | 建立对话管理 | — |
| 4 | 传输层 | 端到端可靠送达 | TCP/UDP |
| 3 | 网络层 | 跨网络路由 | IP |
| 2 | 链路层 | 同网段邻居通信 | MAC/ARP |
| 1 | 物理层 | 电信号/光信号 | 网线/光纤 |

## TCP 三次握手

1. 客户端→服务端：`SYN`（在吗，序号 x）
2. 服务端→客户端：`SYN+ACK`（在的，x+1 收到，我的序号 y）
3. 客户端→服务端：`ACK`（收到 y+1，可以传数据了）

**为什么三次？** 防止旧连接请求浪费服务器资源。两次握手让服务器无法判断请求是否有效。

## tcpdump 抓包实操

### 工具

- `tcpdump` — Linux 命令行抓包（和 Wireshark 同一底层库 libpcap）
- `nc` (netcat) — 网络瑞士军刀，可当客户端/服务端

### 命令速查

```bash
# 安装
sudo apt install -y tcpdump

# 看网卡
ip link show       # 所有网卡
ip route           # 看默认走哪张

# 抓包
sudo tcpdump -i lo -n port 9999        # 抓 loopback 指定端口
sudo tcpdump -i enp0s3 -c 10 -n port 80  # 抓外网 10 包

# nc 服务端/客户端
nc -l -p 9999                # 服务端，监听 9999
echo "hi" | nc -q 0 127.0.0.1 9999  # 客户端，发完即断
```

### 抓包三步走（loopback 自测）

```bash
# 窗口1：起服务端
nc -l -p 9999

# 窗口2：抓包
sudo tcpdump -i lo -n port 9999

# 窗口3：发请求
echo "hi" | nc -q 0 127.0.0.1 9999
```

### 实战抓包解读

```
# === 三次握手 ===
127.0.0.1.51444 > 127.0.0.1.9999: Flags [S]         ← SYN（客户端）
127.0.0.1.9999  > 127.0.0.1.51444: Flags [S.]        ← SYN+ACK（服务端）
127.0.0.1.51444 > 127.0.0.1.9999: Flags [.]          ← ACK（握手完成）

# === 数据传输 ===
127.0.0.1.51444 > 127.0.0.1.9999: Flags [P.], seq 1:4  ← 推送 3 字节数据
127.0.0.1.9999  > 127.0.0.1.51444: Flags [.], ack 4     ← 收到确认

# === 四次挥手 ===
127.0.0.1.51444 > 127.0.0.1.9999: Flags [F.]         ← FIN（客户端请求断开）
127.0.0.1.9999  > 127.0.0.1.51444: Flags [.]          ← ACK（服务端确认接收）
127.0.0.1.9999  > 127.0.0.1.51444: Flags [F.]         ← FIN（服务端也断开）
127.0.0.1.51444 > 127.0.0.1.9999: Flags [.]          ← ACK（确认断开）
```

### Flags 速记

| Flag | 含义 | 作用 |
|------|------|------|
| `[S]` | SYN | 发起连接 |
| `[S.]` | SYN+ACK | 确认接收并发出自己的序列号 |
| `[.]` | ACK | 确认收到 |
| `[P.]` | PSH+ACK | 推送数据 |
| `[F.]` | FIN+ACK | 请求断开 |
| `[R.]` | RST+ACK | 强制拒绝/重置连接 |

### 关键概念

- **MAC vs IP**：MAC 是身份证号（出厂固定，链路层局域网认人），IP 是地址（可变，网络层找路）
- **`0.0.0.0:PORT`**：监听所有网卡的所有 IP，127.0.0.1、192.168.x.x、10.0.2.2 都能命
- **RST 不是被拦截**：`[R.]` 说明对方收到了但主动拒绝（如端口未监听）；被防火墙 DROP 是沉默超时
- **`-q 0`**：nc 发完数据立刻断开，否则 nc 会蹲着等对方回复
