## 必背命令模板（默写四件套）

```bash
# 1. 统计 /var/log 下 7 天内修改、大于 1M 的 .log 文件数
find /var/log -name "*.log" -mtime -7 -size +1M | wc -l

# 2. 每天凌晨 2:30 执行脚本，输出和报错都追加到日志
30 2 * * * /opt/clean.sh >> /var/log/clean.log 2>&1

# 3. 从访问日志筛出 404 请求的来源 IP 并去重
grep ' 404 ' access.log | awk '{print $1}' | sort -u

# 4. 磁盘排查：先看哪个分区满，再逐层找大目录
df -h
du -sh /var/* | sort -hr | head
```

记忆点：

- find 参数全带横杠成对出现：`-name 啥名 -mtime 几天 -size 多大`；只有 mtime 有 m（m=modify）
- crontab 五位口诀：**分 时 日 月 周**
- 管道三步流水线：**筛行(grep) → 切列(awk) → 去重(sort -u)**，顺序不能换
- `' 404 '` 两边空格锚定状态码列，防止误伤 `/404.html`

## HTTP 状态码：先框架后举例

**2xx 成功、3xx 重定向、4xx 客户端错、5xx 服务端错**

- 200 成功｜301 永久重定向（网址搬家）｜404 资源不存在｜500 服务器内部错误
- 502/504 = 反向代理活着，它身后的应用挂了（排查利器）

## 网站打不开：排查链

```
本机网络(能开其他网站吗) → DNS(nslookup 能解析吗) → ping(机器活着吗)
   ├─ ping 不通 → traceroute 看断在第几跳（第1跳=自家路由，中间=运营商）
   └─ ping 通 → curl/telnet 端口(服务活着吗)
                  ├─ 连接拒绝 = 服务挂/端口没开
                  ├─ 502/504 = 代理活着，后端挂了
                  └─ 上服务器 systemctl status + 看日志
```

要点：ping 只测机器（ICMP，第 3 层，无端口概念），ping 通不代表服务活着。

## TCP vs UDP 一句总纲

**TCP 面向连接、可靠、有序（三次握手+丢包重传）；UDP 无连接、不保证送达、但快。**
例子：TCP → HTTP/SSH；UDP → DNS/视频/游戏。

## SQL 注入

- 原理：用户输入被**拼进 SQL 当代码执行**。经典输入 `admin' --` 把密码判断整个注释掉
- 防御：**参数化查询（预编译）**——SQL 模板先定型，用户输入永远只是"填空的数据"，成不了代码
- 类比：`printf("%s", input)` 安全，`system(input)` 危险

## 安全体系四件套

```
等保测评 = 国家安全年检（按标准清单查配置和制度，三级每年一测）
堡垒机   = 机房唯一的门 + 全程录像（身份在堡垒机层区分，操作可追溯到人）
SIEM     = 全公司日志汇到一块大屏，关联分析自动告警（工具）
SOC      = 7x24 盯大屏的值班团队：研判真假 → 处置 → 上报（人）
```

- 等保 vs SOC：**年度体检 vs 病房护士站**
- 堡垒机为什么不可替代：共用 root 也能追溯到人；审计日志存在堡垒机上，服务器 root 删不掉；离职禁一个账号全线收权
- SIEM 灵魂是**关联分析**：50 条登录失败+1 条成功 = 爆破得手，单条日志看不出来

## SSH 密钥登录

- 公钥=锁（放服务器 `~/.ssh/authorized_keys`），私钥=钥匙（自己保管）
- 登录时服务器用公钥出题，私钥才解得开，**全程不传输密码**
- 强在：不可爆破、抓包无秘密、每人一对天然区分身份
- 加固组合：`PasswordAuthentication no` + `PermitRootLogin no` + 白名单 IP + 堡垒机
